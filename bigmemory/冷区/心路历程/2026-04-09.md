# 2026-04-09 心路

## bigmemory 设计三次迭代的教训
从 v1(hook sub-agent 全自动)→ v2(纯 bash dump)→ v3(按需拉取主 AI 操作)→ 最终版(自动派 worker + 手动归档),每次迭代都是因为 Dr Sun 指出了一个我没考虑到的用户视角问题:
1. "sub-agent 操作人看不到" — 透明性比自动化更重要
2. "直接 dump 是一坨屎山" — 需要智能过滤,不能无脑注入
3. "应该用户第一句话后自动抓,结束时我自己手动归档" — 入口自动,出口手动,给用户控制权

核心认知:记忆系统的设计不是技术问题,是**谁有控制权**的问题。入口可以自动(因为只是读取),出口必须手动(因为涉及写入和判断什么值得保留)。

## Claude Code 与 Droid 统一的局限
一开始以为同步文件就能统一体验,但 Dr Sun 反复提醒"不在这"。最终认清:
- **可统一的**: commands、skills、droids/agents、hooks、settings.json、CLAUDE.md≡AGENTS.md
- **不可统一的**: /compact、内置 memory、context 可视化、平台 UI 行为

这不是配置能解决的问题,是两个平台的产品决策不同。
