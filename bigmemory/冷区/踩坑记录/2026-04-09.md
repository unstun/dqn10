# 2026-04-09 踩坑

## bigmemory 原版 sub-agent 黑箱问题
- **现象**：bigmemory 入口/出口 Sonnet sub-agent 在 hook 里静默运行,用户完全看不到读了什么、写了什么
- **原因**：hook 的 stdout 在 Claude Code 中只在 transcript mode (Ctrl-R) 可见,普通对话界面不显示;Droid 的 SessionStart hook stdout 注入 context 但用户也看不到
- **解决**：去掉入口/出口 sub-agent,改为主 AI 在对话中透明操作
- **预防**：hook 里不要放有副作用的 AI 操作,只做简单数据输出

## Factory Droid sub-agent 看不到 AGENTS.md
- **现象**：以为 memory-worker droid 会自动继承 AGENTS.md 的 35 条硬规则
- **原因**：Factory 文档明确说 "Context isolation — each subagent runs with a fresh context window",droid 只看自己 .md 文件里的 prompt
- **解决**：需要 droid 遵守的规则必须写进 droid 定义文件本身
- **预防**：创建 droid 时把关键约束直接写入 droid prompt

## Claude Code vs Droid 平台功能差异
- **现象**：以为同步文件(.claude/ → .factory/)就能统一体验
- **原因**：两个平台的内置功能不同(/compact、内置 memory、context 可视化等是平台能力,不是配置文件)
- **解决**：接受平台差异,能同步的配置文件同步,平台原生功能无法跨平台复制
- **预防**：区分"可配置的"(commands/skills/droids/hooks)和"平台内置的"(compact/memory/context显示)
