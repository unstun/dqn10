# 用户偏好

> 最后更新：2026-04-09（更新三阶段目录结构路径）

## 交互

- 默认中文回复,思考语言为英语
- 回复以"Dr Sun,"开头
- 改文件前输出 3-7 步计划 + 文件清单 + 风险 + 验证,等"开始"后再动手
- 不确定的研究决策先问 Dr Sun 而不是自行决定
- 用户质疑时不盲目顺从,回查原文事实再回应

## 工具

- 代码搜索优先 Grep 精确匹配,Glob 做文件发现
- 联网搜索用 WebSearch,付费墙用 Playwright MCP
- WebFetch 与 WebSearch 不混在同一批并行(403 级联)
- PDF 链接大概率失败,优先 HTML 版本(如 arxiv.org/html/)
- 复杂任务默认多 Agent 并行

## 代码

- 代码包名 `ugv_dqn`,import 用 `from ugv_dqn.xxx import ...`
- 注释 ASCII 风格分块,代码如顶级开源库
- 所有训练/推理参数通过 `2_experiment/configs/*.json` 管理
- 代码改动须新增 `2_experiment/configs/repro_YYYYMMDD_<topic>.json`

## 论文写作

- 先中文写作,定稿后统一英文润色
- 禁止括号补充说明(缩写定义除外),用"即""由…构成"替代
- 公式用标准数学符号($\Delta c$, $\delta$, $\epsilon$),禁止代码风格变量名
- 禁止"张量""编码"描述地图输入,用"地图""记录"替代
- 方法论不写 enumerate 列表,须散文叙事
- 禁止捏造术语/过度包装/推销性语言

## Git

- 改文件前先 git push 备份(无例外)
- 每完成一个有意义变更就 commit
- 远端训练前必须 rsync 同步 `2_experiment/`（含 configs/ + ugv_dqn/ + scripts/）

## Harness 维护

- `.claude/` 为 commands/skills 单一来源，`.factory/` 下对应目录用 symlink 指向 `.claude/`，symlink 隐式引用可接受
- rsync 到远端时加 `--copy-links` 把 symlink 展开为真实文件
- `.claude/agents/` 和 `.factory/droids/` 格式不同、职责不同，不合并
- `scripts/` 分三层：`run/`（启动脚本）、`analysis/`（分析脚本）、顶层单独文件（如 deploy_server.py）
- `deploy_server.py` 是跨 profile 的统一部署入口，单独保留顶层，不归入任何子目录
- 只使用 Claude Code 和 Droid 两个 AI 平台，不维护其他平台（.agents/ 已删除）
- Harness 维护会话应主动过一遍未关闭决策，避免惰性积累
