# 2026-04-09 调研记录

## 01:00 — Factory Droid 配置体系
- **背景**：需要在 Droid 中复现 Claude Code 的 hooks/commands/skills/agents 体系
- **来源**：
  - https://docs.factory.ai/cli/configuration/custom-droids.md
  - https://docs.factory.ai/cli/configuration/custom-slash-commands.md
  - https://docs.factory.ai/cli/configuration/skills.md
  - https://docs.factory.ai/reference/hooks-reference.md
  - https://docs.factory.ai/cli/user-guides/choosing-your-model.md
  - https://docs.factory.ai/pricing.md
- **关键发现**：
  - Droid hooks 在 `.factory/settings.json`,格式与 Claude Code 完全一致
  - Droid droids 在 `.factory/droids/*.md`,YAML frontmatter + markdown body
  - Droid commands 在 `.factory/commands/*.md`,格式与 Claude Code 一致
  - Droid skills 在 `.factory/skills/`,格式兼容 Claude Code(额外字段被忽略)
  - GPT-5.4 model ID: `gpt-5.4`(1× multiplier)
  - Claude Sonnet 4.6 model ID: `claude-sonnet-4-6`(1.2× multiplier)
  - Sub-agent 有 context isolation,看不到 AGENTS.md
- **结论**：文件格式高度兼容,可直接复制;但平台内置功能(compact/memory/context)无法跨平台

## 01:30 — Claude Code 完整配置体系
- **背景**：对比 Claude Code 有什么 Droid 没有
- **来源**：https://claudefa.st/blog/guide/settings-reference
- **关键发现**：
  - 5 层设置优先级(managed→CLI→local→project→user)
  - 内置 memory 在 `~/.claude/projects/.../memory/`
  - `CLAUDE.md` 支持 user/project/local 三级
  - SessionStart hook stdout 在 Claude Code 普通界面不显示(只在 transcript Ctrl-R)
- **结论**：平台层差异无法通过项目配置弥补
