# 记忆文件格式规范 — DQN10

## 热区容量预算

| 文件 | 上限 | 内容约束 |
|------|------|----------|
| 状态简报.md | 1500字 / 30行 | 活跃任务≤3条,关键上下文≤5条 |
| 未关闭决策.md | 1200字 / 25行 | 未关闭≤15条,已关闭保留7天后删除 |
| 近期改动.md | 1000字 / 25行 | 7天滚动窗口 |

热区总预算约 3700 字。每次更新后须在预算内。
热区是**全量重写**,不是追加。

## 热区文件格式

### 状态简报.md

```markdown
# 项目状态简报
> 最后更新：YYYY-MM-DD HH:MM

## 当前在做什么
- [任务描述]（≤3条）

## 关键上下文
- [影响当前工作的重要背景]（≤5条）

## 近期警告
- ⚠ [需要注意的事项]
```

### 未关闭决策.md

```markdown
# 未关闭决策

- [ ] [决策描述]（提出日期：YYYY-MM-DD）
- [x] ~~[已关闭决策]~~ → [结论]，YYYY-MM-DD
```

### 近期改动.md

```markdown
# 近期改动摘要
> 滚动窗口：最近 7 天

## YYYY-MM-DD
- [改动摘要]
```

滚动规则:
- 今天/昨天:每天≤3条
- 2-4天前:每天≤2条
- 5-7天前:每天≤1条
- 超过7天:删除

## 冷区按天文件格式

文件命名:`YYYY-MM-DD.md`,同一天多次写入使用追加模式。
每条记录用二级标题分隔:`## HH:MM — 简述`

### 改动记录/YYYY-MM-DD.md

```markdown
## HH:MM — [改动简述]
- **原因**：[为什么要改]
- **改动**：[文件路径 + 改了什么]
- **训练/推理**：[如启动了训练/推理,记录 profile、预计时长、远端路径]
- **状态**：[进行中/已完成/已回滚]
```

质量门槛:只记录改变系统行为或架构的改动,不记录纯格式调整。

### 踩坑记录/YYYY-MM-DD.md

```markdown
## [坑的简述]
- **现象**：[看到了什么]
- **原因**：[根因分析]
- **解决**：[解决方案/命令]
- **预防**：[如何避免再次踩坑]
```

质量门槛:只记录调查超过 5 分钟或根因非显而易见的问题。

### 调研记录/YYYY-MM-DD.md

```markdown
## HH:MM — [调研主题]
- **背景**：[为什么要调研]
- **来源**：[URL / 论文 / 文档]
- **关键发现**：[发现列表]
- **结论**：[调研结论]
```

质量门槛:必须包含至少一个具体的 URL 或文献标识。

### 心路历程/YYYY-MM-DD.md

```markdown
## [主题]
[思路变化、决策原因、反思、对项目方向的想法]
```

质量门槛:必须包含"为什么",纯"做了什么"并入改动记录。

### 里程碑/YYYY-MM-DD.md

```markdown
## [里程碑名称]
- **达成内容**：[具体完成了什么]
- **关键指标**：[数据/效果]
- **下一步**：[后续方向]
```

### 会话记录/YYYY-MM-DD.md

```markdown
## HH:MM — [Claude Code / Droid / Codex] [会话主题]

**Dr Sun**: [原话]
**AI**: [回复，保留推理链和关键细节，跳过纯工具调用输出]

**Dr Sun**: ...
**AI**: ...

> 产出：[文件/commit 列表，可选]
```

质量门槛:保留对话原貌——Dr Sun 原话尽量原样,AI 回复保留推理和结论,砍掉机械性内容(读文件输出、grep 结果、工具调用细节)。纯执行类会话(如"改个变量名")可跳过。

## 冷区单文件格式

### 偏好.md / 工作流.md

自由格式,按类别分组,每次修改更新顶部日期。
