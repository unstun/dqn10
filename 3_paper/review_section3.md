**原文摘录**（第 280-284 行）：
```latex

  r_1 = k_p \bigl(D_{k-1} - D_k\bigr)
\end{equation}
其中$D_k$表示第$k$步的测地线目标距离。这种基于势函数的奖励塑形为智能体提供了
具有障碍物感知能力的方向引导，同时不改变最优策略。
```

**审稿意见**：
1. 声称"不改变最优策略"但未引用 Ng et al. (1999) 的 PBRS 理论（注：references.bib 已有 `ng1999policy` 条目但正文未引用）。审稿人会质疑：凭什么断言不改变最优策略？需要给出势函数定义并说明满足 PBRS 条件。

3. r₄ 安全距离惩罚的函数形式选择未解释——审稿人会问"为什么用反距离函数而不是线性或二次函数？"

**建议修改**：

#### 1a. r₁ 进度奖励（替换第 280-284 行）

```latex
\textit{A.~进度奖励$r_1$}

为在保证最优策略不变的前提下提供稠密方向引导，本文基于势函数奖励塑形
（PBRS）理论~\cite{ng1999policy}设计进度奖励。定义势函数
$\Phi(s) = -D(s)$，其中 $D(s)$ 为状态 $s$ 到目标的测地线距离，
则进度奖励为相邻状态势函数之差：
\begin{equation}\label{eq:reward_prog}
  r_1 = k_p \bigl(\Phi(s_{t+1}) - \Phi(s_t)\bigr)
      = k_p \bigl(D_t - D_{t+1}\bigr)
\end{equation}
由于 $r_1$ 严格满足 $F(s,a,s')=\gamma\Phi(s')-\Phi(s)$ 的
PBRS 形式（此处取 $\gamma\approx1$），该塑形奖励不改变最优策略，
同时通过 Dijkstra 距离场提供具有障碍物感知能力的全局方向梯度。
```

```latex
曲率惩罚三项组成：
```


### 问题 9：缺少算法伪代码

**审稿意见**：整个方法章节没有 Algorithm 环境。对于一个包含 DQfD 预训练 + 在线训练 + 动作掩码 + 多组件联动的系统，缺少伪代码会显著降低可读性和可复现性。建议至少增加 1 个 Algorithm 来概述完整流程。

**建议修改**：在动作掩码节末尾（第 443 行后）增加。

```latex
\begin{algorithm}[!htbp]
\caption{MD-DDQN 训练与推理流程}
\label{alg:training}
\begin{algorithmic}[1]
\Require 占据栅格地图 $\mathcal{G}$，Dijkstra 距离场 $D$，
         专家演示集 $\mathcal{D}_E$
\Ensure  训练后的策略网络 $Q_\theta$

\State \textbf{// 阶段一：DQfD 预训练}
\State 将 $\mathcal{D}_E$ 填充至回放缓冲区 $\mathcal{R}$
\For{$i = 1$ \textbf{to} $N_{\mathrm{pre}}$}
    \State 从 $\mathcal{R}$ 采样小批量 $\mathcal{B}$
    \State 计算复合损失
           $\mathcal{L}_{\mathrm{pre}} = \mathcal{L}_{\mathrm{TD}}
           + \lambda_{\mathrm{M}}\mathcal{L}_{\mathrm{margin}}
           + \lambda_{\mathrm{CE}}\mathcal{L}_{\mathrm{CE}}$
    \State 更新 $\theta \leftarrow \theta - \eta \nabla_\theta \mathcal{L}_{\mathrm{pre}}$
\EndFor

\State \textbf{// 阶段二：在线训练}
\For{episode $= 1$ \textbf{to} $N_{\mathrm{ep}}$}
    \State 随机采样起终点，初始化状态 $s_0$
    \While{未终止}
        \State 计算可接纳掩码
               $\mathbf{m}(s_t) \leftarrow \textsc{ActionMask}(s_t, \mathcal{G}, D)$
        \If{$\mathrm{rand}() < \varepsilon$}
            \State $a_t \sim \mathrm{Uniform}(\mathcal{A}_{\mathrm{adm}}(s_t))$
            \Comment{探索}
        \Else
            \State $a_t \leftarrow \arg\max_{a \in \mathcal{A}_{\mathrm{adm}}(s_t)} Q_\theta(s_t, a)$
            \Comment{利用}
        \EndIf
        \State 执行 $a_t$，观测 $(s_{t+1}, r_{t+1})$
        \State 存储 $(s_t, a_t, r_{t+1}, s_{t+1}, \mathbf{m}(s_{t+1}))$
               至 $\mathcal{R}$
        \State 采样 $\mathcal{B}$，以掩码 TD 目标（公式~\ref{eq:masked_target}）
               计算 $\mathcal{L}$，更新 $\theta$
        \State 每 $T_{\mathrm{target}}$ 步同步
               $\theta^{-} \leftarrow \theta$
    \EndWhile
\EndFor
\end{algorithmic}
\end{algorithm}
```

> **注意**：上述伪代码中的 `\textsc{ActionMask}` 对应论文第 416-433 行描述的可行性评估流程。如果需要，可以将动作掩码的细节也写成一个独立的 Algorithm 2。
